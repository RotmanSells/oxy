# Структура проекта

Проект разложен так, чтобы в корне оставались только точки входа и крупные рабочие зоны.

## Корень

- `index.html` — основной лендинг.
- `css/` — стили лендинга.
- `js/` — клиентская логика и переводы.
- `api/` — backend для заявок и админки.
- `assets/` — production-медиа лендинга.
- `docs/` — документация проекта.
- `scripts/` — локальные утилиты обработки ассетов.
- `security-review/` — карантин для security-артефактов, не для production.

## Assets

- `assets/images/site/` — изображения, которые напрямую использует лендинг.
- `assets/images/featured/` — выделенные изображения для hero/project/lightbox-блоков.
- `assets/videos/site/` — видео, которые напрямую использует лендинг.
- `assets/videos/featured/` — выделенные видео для ключевых блоков.

## Правила добавления файлов

- Новые production-изображения класть в `assets/images/site/` или `assets/images/featured/`.
- Новые production-видео класть в `assets/videos/site/` или `assets/videos/featured/`.
- Исходники, промежуточные версии и эксперименты не хранить в корне; при необходимости держать вне production-репозитория или в отдельно согласованном архиве.
- Документы добавлять в `docs/`, скрипты — в `scripts/`.
- Не класть медиа, `.DS_Store`, временные файлы и security-скрипты в корень проекта.

## Проверка после переносов

```bash
node --check js/main.js
node --check js/translations.js
```

После изменения путей дополнительно проверить страницу локально через статический сервер.
